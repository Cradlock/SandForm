## class ResourceManager
Ресурсный менеджер через него можно читать и создавать ресурсы.
Создает свой поток (для загрузки файлов).Менеджер умеет кешировать данные.
Также отслеживать  изменения в реальном времени 
(если включена настройка hot reload).

--- 
В ResourceManager файлы будут хранится как 
обьект `IResource` от него будут наследоватся
другие обьекты

* Texture -> render module  

* Sound -> sound module 

* Json -> ... modules 

... И другие структуры. 


`IResource` - уникальный интелектуальный 
посредник, обозначающий информацию о файле
и статус его сушествования.




Для отдельных модулей мы просто отправляем 
функции. Модули вообще не знают что есть классы
и разные функции они просто получают указатель
на обьект ресурса и набор функции для них.
Это позволяет разным языкам работать с моим `core`
модулем.

Система загрузки включается в себя новый поток.
Отвечающий за загрузку файлов из очереди.


---

Типы файлов: 
> Файлы делятся по способу обработки в памяти 

### Статические 
Ресурсы которые загружаютс полностью и не 
меняются при работе.(Модели,Текстуры)


### Сценарные и конфигурационные
Эти файлы после загрузки превращаются в 
древовидные структуры или байт-код.
(Json,YAML,Lua, ...)

### Потоковые 
Ресурсы которые никогда не загружаются целиком.
А только частями.(Длинные аудио,видео)
