# Основной центр от которого все зависимы

Модуль который будут использовать все остальные,мозг игры.

---

В самом модуле есть несколько главных обьектов:

### class Engine 
> Основной двигатель 

### class Eventbus 
> Общение между модулями

### class ResourceManager
> Менеджер ресурсов

### class Time 
> Нужен для системы тиков

### class Config 
> Менеджер настроек (push model)

### class Logger 
> Система логирования

### namespace interfaces
> Контракты между модулями 

---

## class Logger  
Создает в папке `logs/` файлы:

- warning.log 

- error.log 

- info.log   

- fatal.log 


После записывается в конец этого файла как 
новая строка,а с ней и дата.

`Logger`  - создает отдельный поток чтобы избежать фризов при записи не критичных логов.
например `info` или `warning`:


## class ResourceManager
Ресурсный менеджер - общий класс для контроля ресурсов и работы с файлами.

---

Все ресурсы хранятся ввиде обьектов:

### ResourcInternal 
```
class ResourceInternal{
  std::atomic<int> ref_count;
  fs::path path;
  string extension;
  const ResourceVtable* vtable;
  ResourceState state;
  
  Resource pub;
  
  static ResourceInternal from_res(Resource*);
};
```

Внутренний обьект который нужен для работы в самом ресурс-менеджере,
не должен выходит за ее пределы

### Resource 
```
struct Resource{
 void* data;
   
};
```
Публичный обьект - создан для C ABI чтобы разные языки могли понять структуру,


### void* data 
```
void* data; 
```
Данные универсальные для каждого модуля какждый модуль сам решает что он там себе запишет;




## class EventBus 
Система общения между модулями по схеме `шины событий`.
`class EventBus` - основной класс, который выдает функции и регистрирует новые 
событий, модуль логики при инициализации создает все типы собыйти и нужные 
структуры для функций. (все это будет прописано в .h интерфейсе).

### Функции API

- `subscribe(eventname,callback)`: Подписка на событие, 
возвращает токен определенного события

- `unsubscribe(token)`: Отписка от события

- `publish(name,data)`: Функция запускаюшая события
